Fixing the network device names

12 October 2017 - less than 1 min read time
Tags: Debian Linux Recomendations Exherbo ArchLinux

If you are struggled with a distro with systemd for a random reason, you can fix this dumb nonsense of network devices with this command:

ln -s /dev/null /etc/systemd/network/99-default.link

I don’t know what will break doing this, but since I’m testing this using a VM, I don’t really care (but you should, so beware where are you doing this). I’ve tested on Debian and Arch. Some people reported that most problems with network manager can be fixed when doing this.

Update: You can also fix this in non-systemd distros using this in your boot:

net.ifnames=0 biosdevname=0
